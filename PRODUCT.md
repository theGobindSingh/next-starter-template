## Product Purpose

This starter template exists to give developers a batteries-included Next.js foundation
with an opinionated structure, modern tooling, and a striking default look. Success means
a developer can clone it, run `pnpm install && pnpm dev`, and immediately feel "this is
professional" — then modify every token to make it their own.

## Anti-references

- Generic AI-designed websites — flashy gradients everywhere, glassmorphism by default,
  over-rounded cards, identical icon-card grids, section eyebrow kickers ("ABOUT" /
  "PROCESS" / "PRICING" in tracked uppercase above every heading).
- Uniform one-color-everywhere palette — the "startup blue" monoculture or any single
  dominant hue applied without restraint.
- Bland, basic, corporate templates — generic bootstrap-style layouts, flat design
  without hierarchy, no typography system, no rhythm.

## Design Principles

1. **Templateship** — This is a starting point, not a final product. Every color, font,
   and spacing value is a token that end users can override. The default must be
   striking, but the structure must be transparent and easy to customize.

2. **Craft over flash** — Earn attention through typography, rhythm, and deliberate
   restraint — not through gradient overlays, decorative blurs, or gratuitous motion.
   Quality surfaces through precision, not decoration.

3. **No AI slop** — Every component must pass the "would someone say this was
   AI-generated?" test. Avoid the tells: warm-tinted defaults, over-rounded corners,
   side-stripe borders, eyebrow kickers on every section, identical card grids,
   hand-drawn SVG illustrations.

4. **Accessibility is not optional** — WCAG 2.1 AA is the floor. ≥4.5:1 contrast,
   `prefers-reduced-motion` support, semantic HTML, keyboard-navigable interactions,
   screen-reader-friendly labels.

## Accessibility & Inclusion

**Target**: WCAG 2.1 AA minimum
- Body text contrast ≥ 4.5:1; large text (≥18px / bold ≥14px) ≥ 3:1
- All interactive elements keyboard-focusable with visible focus rings
- `prefers-reduced-motion` respected on every animation
- Semantic HTML structure (headings in order, landmark elements, proper alt text)
- Color not used as sole indicator for state or meaning
- Screen reader support: ARIA labels where native semantics are insufficient
