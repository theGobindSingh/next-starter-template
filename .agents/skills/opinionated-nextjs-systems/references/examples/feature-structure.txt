project-root/
  modules/
    feature-name/
      index.tsx           # composition only; imports sections + constants
      constants.ts        # typed static content arrays/objects
      types.ts            # interfaces/types only
      styles.ts           # Emotion styled + css fragments
      hero.tsx            # optional hero section
      overview.tsx        # standard section file
      cta-strip.tsx       # conversion section file

  pages/                  # or app/ if host project uses App Router
    feature-name.tsx      # thin route wrapper importing module root

Composition flow:
constants.ts -> section files -> module index.tsx -> route file

Hard boundaries:
- No static arrays in module index.tsx
- No heavy business logic in route files
- No ad-hoc styles outside styles.ts
- Prefer shared primitives/wrappers over raw duplicated markup
